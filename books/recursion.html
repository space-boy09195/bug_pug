A SLIGHTLY MORE HELPFUL DEFINITION

Recursion is the practice of a function calling itself. This sounds circular because it is. It also sounds dangerous because it is. The key ingredient that prevents infinite chaos is the base case — the condition under which the function stops calling itself and returns a value like a normal, well-adjusted function.

function countdown(n) { // BASE CASE: stop here if (n <= 0) return "🚀 Liftoff!"; // RECURSIVE CASE: call yourself return n + "... " + countdown(n - 1); } countdown(5); // → "5... 4... 3... 2... 1... 🚀 Liftoff!"
"Every recursive function is a story with an ending. Without the base case, it is a story told by someone who never learned to stop talking."

THE CALL STACK: A METAPHOR

Imagine a stack of plates. Every time your function calls itself, a new plate is placed on top. When the base case is hit, plates are removed from the top, one by one, each returning its value to the plate below.

If you never hit the base case, you keep adding plates until the stack falls over. In programming, this is called a Stack Overflow — also a website where you will find the solution to this exact problem, posted by someone who had the same problem in 2011 and has since moved on with their life.

WHEN TO USE RECURSION

✅ Tree traversal (folder structures, DOM, family arguments)

✅ Divide and conquer algorithms (merge sort, quicksort)

✅ Mathematical sequences (factorial, Fibonacci)

✅ When you want to feel smart

❌ When a simple loop would work fine and everyone else has to maintain this

❌ When the depth could be very large (prefer iteration)

❌ When your colleague has already said "please don't"
